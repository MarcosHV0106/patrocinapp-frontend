# Contribuir a PatrocinApp Frontend

1. Crear ramas desde `desarrollo` con prefijos `interfaz/`, `diseno/`, `pruebas/` o `infraestructura/`.
2. Mantener componentes standalone, servicios tipados, guards e interceptores; no agregar estado global sin necesidad.
3. Ejecutar `npm ci`, `npm test` y `npm run build` antes del Pull Request.
4. No versionar `.env`, tokens, contraseñas, `node_modules`, `dist` ni datos personales.
5. Solicitar revisión real de otra cuenta y conservar evidencia reproducible. La rama `main` solo recibe lanzamientos aprobados.
